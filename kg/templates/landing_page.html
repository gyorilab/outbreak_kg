CAPTRS Outbreak KG

This project implements a knowledge graph framework for representing disease outbreak scenarios. The knowledge graph is built by processing disease outbreak alerts and other sources and combines this with ontological information to create a structured representation of outbreak events.

KG API:

  • /v1/alerts(disease:str = None, geolocation:str = None, pathogen:str = None, timestamp:str = None, symptom:str = None, limit:int = None):
    Return the subgraph surrounding the corresponding alerts that match the constraints provided.
  • /v1/alerts/<alert_id>:(alert_id:str)
    Load an alert from the alerts folder and return the content of the txt file.
  • /v1/indicators:(geolocation:str, indicator_filter:str)
    Return indicator data for a given country based on an indicator name string-based filter.
  • /v1/text_relations:(text:str)
    Return direct relationships between terms and alerts in which the text annotation co-occurs.
